# 知识库问答助手 (RAG Document QA)

基于检索增强生成（RAG）技术构建的智能知识库问答系统，支持多种文档格式上传、本地化向量存储和多轮对话，通过 Streamlit 提供友好的 Web 交互界面。

## 🌟 核心功能
- 📄 多格式文档支持：PDF、TXT、Markdown、Word（.docx）
- 🧠 本地化向量存储：基于 FAISS 构建私有知识库，数据不泄露
- 💬 智能问答：结合通义千问大模型，基于文档内容精准回答
- 🔍 来源追溯：问答结果可查看对应的文档片段来源
- 🎨 友好界面：Streamlit 构建的可视化操作界面，支持多轮对话
- 🔧 灵活管理：支持知识库重置、对话历史清除、批量加载文档

## 📋 环境要求
- Python 3.8+（推荐 3.10+）
- 操作系统：Windows/macOS/Linux
- 阿里云通义千问 API Key（需[阿里云百炼平台](https://dashscope.aliyun.com/)申请）

## 🛠️ 安装步骤

### 1. 克隆项目（可选）
```bash
git clone <项目仓库地址>
cd rag-document-qa
```

### 2. 创建并激活虚拟环境
```bash
# 创建虚拟环境
python -m venv venv

# Windows 激活
venv\Scripts\activate

# macOS/Linux 激活
source venv/bin/activate
```

### 3. 安装依赖包
```bash
pip install -r requirements.txt
```

## ⚙️ 配置说明

### 1. 获取 API Key
前往 [阿里云百炼大模型平台](https://dashscope.aliyun.com/) 注册并获取 API Key（DashScope API Key）。

### 2. 配置环境变量
在项目根目录创建 `.env` 文件，填入以下内容：
```env
OPENAI_API_KEY=你的通义千问API Key
```

## 🚀 使用方法

### 1. 启动应用
```bash
streamlit run app.py
```

### 2. 基础操作流程
1. **初始化 RAG 引擎**：启动应用后，在左侧侧边栏点击「初始化/重置 RAG 引擎」
2. **加载文档**：
   - 方式1：上传单个/多个文档（PDF/TXT/MD/Word）
   - 方式2：将文档放入 `docs` 目录，点击「加载 docs 目录所有文档」
3. **开始问答**：在主界面输入框中提出问题，系统会基于知识库内容回答
4. **查看来源**：回答下方可展开「查看来源」，查看问答对应的文档片段
5. **管理操作**：支持清除对话历史、重新初始化引擎等

## 📁 项目结构
```
rag-document-qa/
├── app.py                # Streamlit Web 应用主程序（界面/交互逻辑）
├── rag_engine.py         # RAG 核心引擎（向量库/问答链/大模型调用）
├── document_loader.py    # 多格式文档加载器（处理PDF/TXT/MD/Word）
├── requirements.txt      # 项目依赖清单
├── .env                  # 环境变量配置（API Key）
├── faiss_index/          # FAISS 向量库存储目录（自动生成）
├── docs/                 # 示例文档目录（可自行添加文档）
└── .idea/                # IDE 配置文件（无需关注）
```

## 🧰 核心技术栈
| 模块         | 技术/库                          | 说明                     |
|--------------|----------------------------------|--------------------------|
| 前端交互     | Streamlit                        | 快速构建Web应用          |
| RAG 框架     | LangChain 1.x（LCEL）            | 构建检索增强生成流程     |
| 向量数据库   | FAISS                            | 本地化向量存储与检索     |
| 大模型       | 通义千问（qwen-max）             | 基于阿里云DashScope调用  |
| 文档处理     | PyPDFLoader、python-docx等       | 多格式文档解析           |
| 环境管理     | python-dotenv                    | 环境变量配置             |

## 📝 注意事项
1. API Key 需确保有效且有足够的调用额度，否则会导致问答失败
2. 文档处理时会生成临时文件，程序会自动清理，无需手动删除
3. `faiss_index` 目录存储向量库数据，删除该目录会重置整个知识库
4. 建议将大文件拆分为小文档，提升检索和问答效率
5. 问答效果依赖于文档内容的完整性和相关性

## ❓ 常见问题

### Q1: 初始化 RAG 引擎失败？
A1: 检查 `.env` 文件中的 API Key 是否正确，确保网络能访问阿里云百炼平台，且 API Key 未过期/欠费。

### Q2: 上传文档后问答无相关结果？
A2: 
- 确认文档已成功处理（侧边栏显示「文档块数量」）
- 问题需与文档内容相关，避免提出文档外的问题
- 可尝试调整问题表述，或增加更多相关文档

### Q3: 支持的文档大小限制？
A3: 无硬性限制，但建议单文档不超过100MB，大文档可拆分后上传，提升处理效率。

### Q4: 如何重置知识库？
A4: 点击侧边栏「初始化/重置 RAG 引擎」，会清空现有向量库并重新初始化。
